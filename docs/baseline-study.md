## Frozen rules

Common setup: $1,000 contributed on the first trading day of each month; 5 bps friction on every trade; idle cash accrues the FEDFUNDS-based short rate (engine `accrueCash`). Month-start locks use the state of the month's first trading day, matching engine conventions.

- **B1 QQQ DCA**: every month the full available cash buys QQQ (engine `portfolios.qqq`). Reference plan.
- **B2 Drawdown ladder** (QQQ only, buy-only): at each month start, if the Nasdaq-100 5-day-average drawdown is <= -20%, invest up to 3x the monthly amount from accumulated cash; if <= -10%, invest 2x; otherwise 1x.
- **B3 VIX rule** (QQQ only, buy-only): at each month start, if the VIX 5-day average is >= 30, invest 2x the monthly amount; otherwise 1x.
- **B4 Three-signal standard policy**: the existing main strategy (`runBacktest` `portfolios.signal`), used as-is; not modified.

B2/B3 "Nx" spends out of the plan's accumulated cash (unspent cash keeps earning interest), aligned with the three-signal strategy's cash mechanics for fairness.

Declaration: the four rule sets above were frozen before any study output was inspected. The B2/B3 parameters (-10%/-20%/3x/2x and 30/2x) must not be retuned based on these results; results adverse to any plan are reported as-is.

## Method

- 11 start dates x 5 bps cost; $1,000 contributed on each month's first trading day.
- B1 and B4 come from the engine's `runBacktest` (`portfolios.qqq` / `portfolios.signal`). B2/B3 use a small monthly loop built from the same exported accounting primitives (`emptyPortfolio`, `addContribution`, `accrueCash`, `buyQQQ`, `updateDrawdown`, `valueOf`, `riskStats`).
- B2 locks the month-start `drawdownPct`, B3 the month-start `vix` (5-day averages from `buildStates`), both observed on the month's first trading day and acted on the same day, consistent with the engine's QQQ-sleeve convention.
- Metrics: finalValue (terminal portfolio value), maxDrawdown (unit-NAV peak-to-trough), Sharpe (monthly excess return over the modeled cash rate, annualized by sqrt(12), engine `riskStats`), multiple (finalValue / total contributed).
- Loop validation: a 1x naive loop reproduces the engine's QQQ DCA final value within $0.0000 across all 11 starts.
- Synthetic history: QQQ before 1999-03-10 and TQQQ before 2010-02-11 are synthetic (pre-inception). Rows touching synthetic history are flagged: `sQQQ` affects all plans, `sTQQQ` affects B4 only (B1/B2/B3 hold QQQ only).
